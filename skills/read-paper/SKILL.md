When the user provides a URL or PDF of an economics paper:

1. **Fetch or read it fully** before writing anything — use WebFetch for URLs, Read for PDFs.

2. **Produce this concise summary:**

**What they did**
One paragraph. What question did the paper ask, and what did they do to answer it? Include the data source and empirical strategy (e.g. RCT, IV, diff-in-diff, structural model).

**Main finding**
The headline result in 1-3 sentences. Lead with the number or the direction of the effect if there is one.

**Why it matters**
What does this add to the economics literature? Is there a policy implication?

**Watch out for**
1-2 honest caveats — limitations the authors mention, or anything that should make the reader skeptical.

## Style rules
- Be brief. No fluff, no padding.
- Use plain language. Spell out jargon on first use.
- Do not reproduce the abstract — synthesize, don't copy.
- If the paper is gated or unreachable, say so immediately and ask the user to paste the text.
